# タイピングトレーニングゲーム 仕様書

## 概要

既存のゲームフレームワーク（`/note/game/`）を利用した、4つのモードを持つタイピングトレーニングゲーム。通常のタイピング練習に加え、日本語入力、VimキーバインドとmacのControl+矢印操作を実践的に学べる。

## URL

- `/note/game/typing` - ゲーム詳細ページ
- `/note/game/typing/play` - プレイページ

## 4つのモード

### モード1: 通常タイピング

プログラミング向けのタイピング練習。

**問題の種類:**

- プログラミングキーワード（`const`, `function`, `import`, `export`, `return`, `async`, `await`等）
- コードスニペット（`const app = express()`, `useState(false)`, `git commit -m "fix"` 等）
- ターミナルコマンド（`npm install`, `docker compose up`, `git push origin main` 等）
- 記号重視（`() => {}`, `<div className="">`, `${variable}`, `[...arr]` 等）

**難易度（3段階）:**

| レベル | 名前 | 内容                                | 制限時間 |
| ------ | ---- | ----------------------------------- | -------- |
| 1      | 初級 | 英単語・短いキーワード（3-8文字）   | なし     |
| 2      | 中級 | コードスニペット（10-30文字）       | 60秒     |
| 3      | 上級 | 長文コード・記号混じり（30-80文字） | 60秒     |

**UI:**

```
+------------------------------------------+
| モード: 通常タイピング  難易度: 中級      |
| スコア: 1250  正確率: 94.2%  WPM: 68     |
+------------------------------------------+
|                                          |
|   const [count, setCount] = useState(0)  |  <- お題（灰色）
|   const [count, setCou                   |  <- 入力中（緑=正解、赤=ミス）
|                     ^カーソル             |
|                                          |
+------------------------------------------+
| 残り時間: 42秒  |  正解: 12  ミス: 3     |
+------------------------------------------+
```

**スコアリング:**

- 正解1文字 = 10点
- 記号正解 = 15点（`{`, `}`, `(`, `)`, `[`, `]`, `<`, `>`, `$`, `=`, `=>` 等）
- ミス = -5点
- WPM（Words Per Minute）をリアルタイム表示
- 正確率（Accuracy）をリアルタイム表示

### モード2: 日本語タイピング

日本語のコードコメント、コミットメッセージ、ドキュメント記述を練習するモード。ローマ字入力で日本語を正確に打てるようになることが目標。

**なぜ日本語タイピングが必要か:**

- コードコメントは日本語で書くプロジェクトが多い
- コミットメッセージを日本語で書くチームがある
- Issue/PRの本文やレビューコメントは日本語
- Slack/チャットでの技術的なやりとりも日本語
- JSDoc/Docstringの記述に日本語を使う場合がある

**入力方式:**

- ローマ字入力（IMEを使わず、ローマ字で日本語を入力する方式）
- お題は日本語（ひらがな/カタカナ/漢字混じり）で表示
- プレイヤーはその日本語に対応するローマ字を入力する
- 例: お題「関数を定義する」→ 入力「kannsuwoteigisuru」

**ローマ字変換ルール:**

- 基本の50音（a, i, u, e, o, ka, ki, ku, ke, ko ...）
- 促音: 子音を2回（例: 「切った」→ `kitta`）
- 撥音: `nn`（例: 「関数」→ `kannsuu`）、ただし母音/y前以外は`n`1回でもOK
- 拗音: `kya`, `sya`/`sha`, `tya`/`cha` 等の複数パターンを許容
- 長音: そのまま母音を打つ（例: 「サーバー」→ `sa-ba-` or `saabaa`）

**複数の入力パターンを許容する:**

| 日本語 | パターン1 | パターン2 | パターン3 |
| ------ | --------- | --------- | --------- |
| し     | si        | shi       | ci        |
| ち     | ti        | chi       | -         |
| つ     | tu        | tsu       | -         |
| ふ     | hu        | fu        | -         |
| じ     | zi        | ji        | -         |
| しゃ   | sya       | sha       | -         |
| ちゃ   | tya       | cha       | cya       |
| を     | wo        | o         | -         |

**問題の種類:**

| カテゴリ           | 例                                                         | 難易度 |
| ------------------ | ---------------------------------------------------------- | ------ |
| コードコメント     | 「ユーザー情報を取得する」                                 | 初級   |
| コードコメント     | 「データベースに接続してクエリを実行する」                 | 中級   |
| コミットメッセージ | 「バグを修正」                                             | 初級   |
| コミットメッセージ | 「ログイン機能のバリデーションを追加」                     | 中級   |
| PRタイトル         | 「検索機能の実装」                                         | 初級   |
| PR説明文           | 「パフォーマンス改善のためキャッシュを導入しました」       | 上級   |
| レビューコメント   | 「ここはイミュータブルに書いた方が安全です」               | 上級   |
| Slack/チャット     | 「デプロイ完了しました」                                   | 初級   |
| Slack/チャット     | 「ステージング環境でテストお願いします」                   | 中級   |
| ドキュメント       | 「環境変数の設定手順」                                     | 中級   |
| 技術用語           | 「コンポーネント」「リファクタリング」「マイグレーション」 | 初級   |

**難易度:**

| レベル | 内容                                                    | 文字数目安 |
| ------ | ------------------------------------------------------- | ---------- |
| 初級   | ひらがな/カタカナ中心の短い文（技術用語、短いコメント） | 5-15文字   |
| 中級   | 漢字混じりの文（コミットメッセージ、コメント）          | 15-30文字  |
| 上級   | 長文（PR説明、レビューコメント、ドキュメント）          | 30-60文字  |

**UI:**

```
+------------------------------------------+
| モード: 日本語タイピング  難易度: 中級    |
| スコア: 980  正確率: 91.5%  WPM: 42     |
+------------------------------------------+
|                                          |
|   ユーザー情報を取得する                 |  <- お題（日本語表示）
|   yu-za-jyouhouwo                        |  <- 読み仮名ガイド（薄く表示、ON/OFF可）
|   yu-za-jyouhouwo                        |  <- 入力中のローマ字
|              ^カーソル                    |
|                                          |
+------------------------------------------+
| 残り時間: 38秒  正解: 8  ミス: 2         |
| 読み仮名ガイド: [ON/OFF]                |
+------------------------------------------+
```

**読み仮名ガイド機能:**

- お題の漢字の上（または下）に読み仮名をひらがなで表示
- ON/OFFを切替可能
- 初級ではデフォルトON、中級/上級ではデフォルトOFF

**スコアリング:**

- ひらがな1文字分の正解 = 10点
- カタカナ1文字分の正解 = 10点
- 漢字1文字分の正解 = 20点（ローマ字入力量が多いため）
- 促音・拗音の正解 = 15点
- ミス = -5点
- WPMはローマ字入力ベースで計算

**実装上の注意:**

- IMEを無効化する（`inputMode="latin"`, `autoComplete="off"`）
- `compositionstart`/`compositionend`イベントを無視する
- ローマ字→かな変換テーブルを自前で実装
- 1文字ごとにリアルタイムで正誤判定
- 複数パターン（`si`/`shi`）はどちらも正解として処理

### 技術仕様

**コンポーネント構成:**

- `src/components/games/TypingGame.tsx` - メインコンポーネント
  - モード選択、難易度選択のステートマシン
  - 3モードの切替

**キーイベント処理:**

- `keydown`イベントをリッスン
- Vim/macモードではブラウザのデフォルト動作を`preventDefault()`で無効化
- `event.key`, `event.code`, `event.metaKey`, `event.ctrlKey`, `event.altKey`, `event.shiftKey`で判定
- 注意: ブラウザでは一部のショートカット（Cmd+W, Cmd+Tab等）をキャプチャできない
  → キャプチャ不可能なキーは「画面上のボタンをクリック」方式で代替するか、出題から除外

**キャプチャ不可能なキーの対応方針:**

- `Cmd+Tab`, `Cmd+W`, `Cmd+Q` → 出題はするがキーは仮想キーボードで入力 or 選択式
- `Ctrl+C`（ブラウザのコピー）→ 上書き可能、出題OK
- `Cmd+S`（ブラウザの保存）→ `preventDefault()`で上書き可能

**スタイリング:**

- Tailwind CSS
- ダークテーマ（bg-gray-900ベース、既存ゲームに統一）
- カーソルはborder-left: 2px solid で表現
- 正解文字: text-green-400
- ミス文字: text-red-400 + 下線
- 未入力文字: text-gray-500

**データ:**

- 問題データは`src/lib/games/typing-data.ts`に定義
- カテゴリ別、難易度別に配列で管理
- ランダム出題

**スコア保存:**

- localStorageにハイスコア、統計情報を保存
- キー: `typing-game-stats`
- 内容: `{ highScore: number, totalGames: number, avgWPM: number, avgAccuracy: number, weakKeys: { [key: string]: { total: number, miss: number } } }`
